## Getting Started

DeviceRunners provides multiple ways to run tests for .NET MAUI applications:

### Visual Test Runners
Interactive test execution within the IDE for development and debugging. Perfect for manual testing and development workflows.

- **[Visual Runner in the IDE](articles/visual-runner-in-the-ide.md)** - Run tests interactively within your development environment

### CLI Test Runners
Automated command-line test execution for CI/CD and scripted testing scenarios.

#### DeviceRunners CLI (Recommended)
The modern, unified CLI tool that replaces platform-specific scripts:
- **[Using DeviceRunners CLI](articles/using-devicerunners-cli.md)** - Overview and installation
- **[Android CLI Testing](articles/cli-device-runner-for-android-using-devicerunners-cli.md)** - Android APK testing
- **[Windows CLI Testing](articles/cli-device-runner-for-windows-using-devicerunners-cli.md)** - Windows MSIX and EXE testing
- **[macOS CLI Testing](articles/cli-device-runner-for-macos-using-devicerunners-cli.md)** - Mac Catalyst testing

#### XHarness (Legacy)
Platform-specific tools for specialized scenarios:
- **[Using XHarness](articles/using-xharness.md)** - Legacy XHarness-based testing
- Platform-specific XHarness guides for iOS, Android, Mac Catalyst, and Windows

### Architecture and Development
- **[Technical Architecture Overview](articles/technical-architecture-overview.md)** - Comprehensive technical documentation
- **[Types of Tests](articles/types-of-tests.md)** - Understanding different testing approaches
